**Part A — CSS: 글래스 컨테이너 확인/폴리시 (CHRT-01)**

chart-cards.css에서 `.market-chart-card`를 확인한다:
1. dark mode: `backdrop-filter: blur(12px)` 있어야 함 (최외곽 컨테이너이므로 OK)
2. `background: var(--mp-chart-card-bg)` 사용 확인 (custom.css에서 선언된 토큰)
3. `border`, `box-shadow` 글래스 토큰 사용 확인
4. 내부 `.chart-box`에 `backdrop-filter`가 없는지 확인 — 있으면 제거 (내부 카드 backdrop-filter 금지 원칙)

만약 `.market-chart-card`가 이미 올바르면 그대로 유지. 토큰 미사용 부분만 수정.

**Part B — params.toml: pink/rose 드리프트 수정 + 동기화 문서 코멘트 (CHRT-02)**

연구에서 확인된 드리프트:
- CSS `--mp-neon-rose: #F43F5E` vs params.toml `palette.pink = "#FF3366"`

`#FF3366`은 실제로 CSS에서 `--mp-neon-pink`이다 (rose와 다른 변수). 차트에서 `COLORS.pink`는 down/danger 색상으로 사용되므로, CSS의 `--mp-neon-rose`와 일치시키는 것이 맞다.

params.toml에서 `[market_pulse.charts.palette]` 섹션의 `pink` 값을 `"#F43F5E"`로 변경한다.

또한 팔레트 섹션 위에 동기화 참조 코멘트를 추가한다:

```toml
# Chart palette — MUST stay in sync with assets/css/custom/custom.css CSS variables
# primary = --mp-neon-purple, cyan = --mp-neon-cyan, pink = --mp-neon-rose,
# green = --mp-neon-green, yellow = --mp-neon-yellow, blue = --mp-neon-blue
# success_light = --mp-color-up (light), danger_light = --mp-color-down (light)
```

**Part C — CSS: 라이트 모드 최적화 확인 (CHRT-03)**

chart-cards.css의 `:root:not(.dark) .market-chart-card` 규칙을 확인한다:
1. `background: #ffffff` (또는 적절한 라이트 값)
2. `backdrop-filter: none` (라이트 모드에서 blur 불필요)
3. `border`, `box-shadow`가 라이트 모드에 맞는 값

이미 올바르면 유지. 누락된 부분만 추가.
